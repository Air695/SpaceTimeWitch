链接时空卡池添加及配置说明
================================

一、概述
--------

链接时空（TagRelic）卡池系统通过 TagRelicRegistry 集中管理所有标签遗物的元数据，
包括所属卡池组（Group）、等级（Tier）、标签类别（Class）、升级路径和权重。
配置页通过 NTagRelicPoolSettingsControl 自动读取注册表生成 UI，无需额外编码。

运行时，TagRelicConfig 从玩家配置（SpaceTimeWitchSettings）读取有效值，
未配置则回退到 TagRelicRegistry 中的默认值。

配置持久化在 settings.json 中，由 ModDataStore（SaveScope.Global）管理。


二、新增卡池及遗物
-------------------

### 2.1 在 TagRelicRegistry 注册元数据

编辑 Relics/TagRelic.cs，在 Entries 字典中添加：

    [typeof(你的遗物类)] = new(
        CardTags.xxx,      // CardTag（Cards/CardTags.cs 中定义）
        "卡池组名",         // Group，如 "EldenRing"
        1,                  // Tier（1=基础，2/3=进阶）
        "标签类别",         // Class，如 "Attack"、"Debuff"、"Adaptive"
        NextTierTypes: [typeof(升级目标)],   // 线性升级路径（可选）
        Weight: 1.0),       // 出现权重（可选，默认 1.0）

### 2.2 创建遗物类

在对应 Extension 目录下创建：

    [RegisterRelic(typeof(SpaceTimeWitchExRelicPool))]
    public class 你的遗物类 : TagRelic
    {
        public override RelicAssetProfile AssetProfile => new(
            IconPath: "res://images/Extension/Relics/xxx.png",
            IconOutlinePath: "res://images/Extension/Relics/xxx.png",
            BigIconPath: "res://images/Extension/Relics/xxx.png"
        );
        // 遗物效果...
    }

### 2.3 添加遗物本地化名称

在 SpaceTimeWitch/localization/zhs/relics.json（和 eng/relics.json）中添加：

    "SPACE_TIME_WITCH_RELIC_{SNAKE_CASE}.title": "显示名称"

其中 SNAKE_CASE 为类名的 UPPER_SNAKE_CASE 形式：
  ERZ1                     → ERZ1
  STWWoodenPickaxe         → STW_WOODEN_PICKAXE
  STWBookofPierre          → STW_BOOKOF_PIERRE

### 2.4 添加 CardTag

在 Cards/CardTags.cs 中定义新的 CardTag 常数。


三、两种升级路径
-----------------

### 3.1 NextTierTypes（线性升级）

适用于只有一条升级路径。ChronoAstrolabe 升级时自动替换。

    NextTierTypes: [typeof(ERZ2)]

### 3.2 NextTierWeights（分支升级）

适用于多条分支可选。ChronoAstrolabe 按权重随机选择。
设置了 NextTierWeights 后，配置页自动显示分支权重滑条。

    NextTierWeights: new Dictionary<Type, double>
    {
        [typeof(分支A)] = 1.0,
        [typeof(分支B)] = 0.5,
    }

注意：只有唯一升级路径时用 NextTierTypes；有多条分支时用 NextTierWeights。
不要同时设置两者中指向相同目标的情况。所有分支权重相加建议至少为 1。


四、配置页自动适配
--------------------

NTagRelicPoolSettingsControl 自动读取 TagRelicRegistry.Entries：

  - 按 Group 分组，每组一个配置区块
  - 组权重滑条（0~5）+ 组去重 ON/OFF 开关 + 组重置按钮
  - Tier1 遗物自动生成出现权重滑条
  - 遗物名自动从 relics.json 本地化表解析
  - NextTierWeights 存在时自动生成分支权重滑条
  - 所有 Class 自动收集，生成标签去重勾选框

新增卡池/遗物时只需在 TagRelicRegistry 添加条目，
配置页和运行时逻辑自动适配，无需修改 UI 代码。


五、本地化
-----------

### 5.1 设置页固定文本

编辑 SpaceTimeWitch/localization/zhs/settings_ui.json 和 eng/settings_ui.json，
键名前缀 TAG_RELIC_POOL.：

  Key                          中文                    English
  ───────────────────────────  ─────────────────────  ──────────────────────────
  .TITLE                       链接时空卡池配置         Linked Space-Time Card Pool Config
  .GROUP_WEIGHT                链接卡池权重             Pool Weight
  .GROUP_DEDUP                 去重（此组每次只抽一个遗物）Dedup (only one relic per pick from this pool)
  .RESET_ALL                   重置为默认值          Reset to default
  .BRANCH_WEIGHT               分支权重       Branch Weight
  .CLASS_DEDUP                 标签去重                 Tag Dedup
  .CLASS_DEDUP_DESC            不会抽到重复的启用标签     No duplicates of enabled tags
  .RESET_DEDUP                 重置为默认值        Reset to default
  .TOGGLE_ON                   ON                      ON
  .TOGGLE_OFF                  OFF                     OFF

### 5.2 遗物名称

遗物名称从 relics.json 的 SPACE_TIME_WITCH_RELIC_{SNAKE_CASE}.title 解析，
回退显示 Type.Name。详见 2.3 节。

------------------

  组名             Tier1 遗物              升级路径                Class

  EldenRing       ERZ1 (环印骑士直剑)       ERZ1 → ERZ2 → ERZ3    Adaptive

七、运行时 API
----------------

在运行时代码中使用 TagRelicConfig 读取有效值（已融合配置覆盖与注册表默认）：

    double w  = TagRelicConfig.GetEffectiveGroupWeight("EldenRing");
    double rw = TagRelicConfig.GetEffectiveRelicWeight(typeof(ERZ1));
    double bw = TagRelicConfig.GetEffectiveBranchWeight(typeof(ERZ1), typeof(ERZ2));
    bool   l  = TagRelicConfig.IsClassLimited("Attack");
    bool   d  = TagRelicConfig.IsGroupDedupEnabled("EldenRing");
    var    hs = TagRelicConfig.GetActiveLimitedClasses();

遗物基类的 Weight / NextTierWeights 属性也已自动从配置读取：

    // TagRelic.Weight        → TagRelicConfig.GetEffectiveRelicWeight(GetType())
    // TagRelic.NextTierWeights → TagRelicConfig.GetEffectiveNextTierWeights(GetType())


八、配置文件结构（settings.json）
----------------------------------

    {
      "TagRelicGroupWeights":   { "EldenRing": 1, "DevilMayCry": 1, ... },
      "TagRelicGroupDedup":     { "EldenRing": false, ... },
      "TagRelicWeights":        { "ERZ1": 1, "DCB1": 1, ... },
      "TagRelicBranchWeights":  { "ERZ1:ERZ2": 1, ... },
      "TagRelicClassDedup":     { "Attack": true, "Debuff": true, ... }
    }


九、注意事项
--------------

  1. 不要在 TagRelicRegistry 中删除已发布的条目，否则旧存档可能崩溃。
     如需废弃，将 Weight 设为 0。
  2. CardTag 必须在 Cards/CardTags.cs 中定义。
  3. 遗物类必须继承 TagRelic 并标记 [RegisterRelic(typeof(SpaceTimeWitchExRelicPool))]。
  4. 修改注册表默认值后，已有用户配置不会自动更新 —— 用户可点击"重置全部为默认值"恢复。
  5. NextTierWeights 仅用于有多个升级分支的遗物；唯一升级路径用 NextTierTypes。
